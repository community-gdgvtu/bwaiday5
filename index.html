Schema Provisioning

Enable AI extensions and create tables in Cloud SQL Studio.

Open Cloud SQL Studio → authenticate with:

  • Username: postgres
  • Database: postgres
  • Password: cloudsql

Step 1 — Enable Extensions

sql — cloud sql studio
CREATE EXTENSION IF NOT EXISTS google_ml_integration CASCADE;
CREATE EXTENSION IF NOT EXISTS vector;

Step 2 — Create Tables

sql
-- Items Table (The "Profile" you swipe on)
CREATE TABLE items (
   item_id   UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   owner_id  UUID,
   provider_name  TEXT,
   provider_phone TEXT,
   title     TEXT,
   bio       TEXT,
   category  TEXT,
   image_url TEXT,
   item_vector VECTOR(768),
   status    TEXT DEFAULT 'available',
   created_at TIMESTAMP DEFAULT NOW()
);

-- Swipes Table (The Interaction)
CREATE TABLE swipes (
   swipe_id  SERIAL PRIMARY KEY,
   swiper_id UUID,
   item_id   UUID REFERENCES items(item_id),
   direction TEXT CHECK (direction IN ('left', 'right')),
   is_match  BOOLEAN DEFAULT FALSE,
   created_at TIMESTAMP DEFAULT NOW()
);

Step 3 — Grant Permission

sql
GRANT EXECUTE ON FUNCTION embedding TO postgres;

Step 4 — Enable ML Integration (Cloud Shell)

bash — replace INSTANCE_NAME
INSTANCE_NAME="<<Your Cloud SQL Instance Name>>"

gcloud sql instances patch $INSTANCE_NAME --tier=db-custom-1-3840

gcloud sql instances patch $INSTANCE_NAME \
    --database-flags=cloudsql.enable_google_ml_integration=on

gcloud sql instances patch $INSTANCE_NAME --enable-google-ml-integration

Step 5 — Grant Vertex AI Role to Cloud SQL Service Account

bash
INSTANCE_NAME="<<Your Cloud SQL Instance Name>>"
PROJECT_ID=$(gcloud config get-value project)

SA_EMAIL=$(gcloud sql instances describe $INSTANCE_NAME \
    --format='value(serviceAccountEmailAddress)')

gcloud projects add-iam-policy-binding $PROJECT_ID \
    --member="serviceAccount:$SA_EMAIL" \
    --role="roles/aiplatform.user"

Step 6 — Register Gemini 3 Flash in Cloud SQL

sql — replace YOUR_PROJECT_ID
CALL google_ml.create_model(
    model_id            => 'gemini-3-flash-preview',
    model_request_url   => 'https://aiplatform.googleapis.com/v1/projects/<<YOUR_PROJECT_ID>>/locations/global/publishers/google/models/gemini-3-flash-preview:generateContent',
    model_qualified_name => 'gemini-3-flash-preview',
    model_provider      => 'google',
    model_type          => 'generic',
    model_auth_type     => 'cloudsql_service_agent_iam'
);
⚠️

Vector Dimension Mismatch — The items table uses VECTOR(768). Stick to text-embedding-005 or your inserts will fail.

⚠️

IAM Propagation Gap — IAM changes can take a few minutes to propagate. If the SQL CALL fails with a permission error, wait a bit and retry.

Checklist

  • items and swipes tables created
  • GRANT EXECUTE on embedding ran OK
  • ML integration flag enabled via gcloud
  • Vertex AI User role granted to service account
  • Gemini 3 Flash model registered in Cloud SQL

Image Storage (GCS)

Create a public Cloud Storage bucket for surplus item photos.

Create a GCS bucket in the same region as your database and grant public read access for instant image rendering in the swipe cards.

Grant Public Access (Cloud Shell)

bash — replace BUCKET_NAME
BUCKET_NAME="<<your-bucket-name>>"

gcloud storage buckets add-iam-policy-binding gs://$BUCKET_NAME \
    --member="allUsers" \
    --role="roles/storage.objectViewer"
ℹ️

Roles needed: Storage Object Viewer (so everyone can see photos) and Storage Object Creator (for demo upload purposes).

⚠️

Bucket Name Uniqueness — Bucket names are a global namespace. If neighborloop-images is taken, add a random suffix like neighborloop-images-xyz42.

⚠️

Region Drag — Keep your bucket in the same region as your Cloud SQL instance and Cloud Run service for maximum performance.

Checklist

Code Walkthrough

How the magic actually works — In-Database AI in action.

Multimodal "Dating Profile" Generation

When a user uploads a photo, Gemini 3 Flash "sees" the item and writes a witty listing automatically:

python — backend
prompt = """
You are a witty community manager for NeighborLoop.
Analyze this surplus item and return JSON:
{
   "bio": "First-person witty dating-style profile bio, max 2 lines",
   "category": "One-word category",
   "tags": ["tag1", "tag2"]
}
"""

response = genai_client.models.generate_content(
   model="gemini-3-flash-preview",
   contents=[
       types.Part.from_bytes(data=image_bytes, mime_type="image/jpeg"),
       prompt
   ],
   config=types.GenerateContentConfig(response_mime_type="application/json")
)

Real-Time In-Database Embeddings

Instead of calling an embedding model in Python and round-tripping data, embeddings are generated inside the INSERT statement:

sql — real-time embedding on insert
INSERT INTO items (
    owner_id, provider_name, provider_phone,
    title, bio, category, image_url, status, item_vector
)
VALUES (
    :owner, :name, :phone, :title, :bio, :cat, :url, 'available',
    embedding('text-embedding-005', :title || ' ' || :bio)::vector
)
💡

Every item is instantly searchable by semantic meaning the moment it's posted — zero application-layer latency.

Semantic Vector Search

Standard keyword search fails if the exact word isn't in the title. Neighbor Loop uses vector similarity — "something to fix my chair" will find a furniture repair kit:

sql — semantic discovery query
SELECT
    item_id, title, bio, category, image_url,
    1 - (item_vector <=> embedding('text-embedding-005', :query)::vector) AS score
FROM items
WHERE status = 'available'
  AND item_vector IS NOT NULL
ORDER BY score DESC
LIMIT 5

The Swipe Loop

  • Swipe Left — Discard the item.
  • Swipe Right — Record a match in the swipes table and show the provider's contact info.

Deploy to Cloud Run

Ship it! One command to deploy the full app.

From inside the project's root folder in Cloud Shell, run:

bash — replace all placeholders
gcloud run deploy neighbor-loop-cloud-sql \
   --source . \
   --region=us-central1 \
   --allow-unauthenticated \
   --network=easy-cloudsql-vpc \
   --subnet=easy-cloudsql-subnet \
   --vpc-egress=private-ranges-only \
   --set-env-vars GEMINI_API_KEY=<<YOUR_GEMINI_API_KEY>>,DATABASE_URL=postgresql+pg8000://postgres:<<YOUR_PASSWORD>>@<<PRIVATE_IP_HOST>>:5432/postgres,GCS_BUCKET_NAME=<<YOUR_GCS_BUCKET>>
ℹ️

Once the command finishes, it will output a Service URL. Copy it and open it in your browser to test the app.

⚠️

Revision Failed / 500 Error — Check Cloud Run logs. Common causes: typo in DATABASE_URL, or the service account lacks GCS permissions.
